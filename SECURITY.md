# Security policy

## Reporting a vulnerability

If you think you've found a security issue in the Auralogs Java SDK, please **do not open a public issue**.

Instead, email [james.c.e.thomas@gmail.com](mailto:james.c.e.thomas@gmail.com) with:

- A description of the issue
- Steps to reproduce
- Any relevant versions / environments

You'll get an acknowledgement within a few business days. Once triaged, we'll coordinate a fix and a disclosure timeline with you.

## Scope

This policy covers the `ai.auralogs:auralogs-core` and `ai.auralogs:auralogs-slf4j` artifacts published from this repository.

For issues with the Auralogs service itself (ingest, dashboard, analysis), please also include that in your report.
